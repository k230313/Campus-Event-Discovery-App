# Project Weekly Plan

This document gives a basic guide for what the team should focus on each week until the project is completed.

## Week 1-2: Problem Research and Project Planning

- identify the main problem
- discuss the project idea
- decide the main users
- define the project goals

## Week 3: Requirements Documentation

- write the software requirements
- list the main features
- identify user needs
- record functional and non-functional requirements

## Week 4: System Architecture Design

- plan the frontend, backend, and database structure
- decide how the system parts connect
- prepare the basic project structure

## Week 5: Database and API Design

- define the database schema
- define REST API endpoints
- define data validation rules
- document the backend structure

## Week 6: Backend Development

- set up MySQL database
- connect the backend to MySQL
- create the `events` table
- replace temporary event data with database data
- build core API routes

## Week 7: Backend Development

- finish CRUD operations for events
- improve validation and error handling
- test backend routes
- clean up backend code

## Week 8: Frontend Development

- start building the frontend pages
- connect the frontend to the backend
- display event data on the website

## Week 9: Frontend Development

- improve the user interface
- add forms for creating or updating events if needed
- test frontend and backend connection

## Week 10: System Integration

- make sure frontend, backend, and database work together
- fix connection issues
- test full system flow

## Week 11: Testing and Deployment

- test important features
- fix final bugs
- prepare the project for submission or deployment
- finalize documentation

## Final Goal

By the end of the project, the team should have:

- a working campus event discovery system
- project documentation
- backend connected to a database
- frontend connected to the backend
- tested core features
