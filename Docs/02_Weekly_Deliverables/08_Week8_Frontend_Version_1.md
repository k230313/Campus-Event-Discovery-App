# Week 8: Frontend Version 1

This document explains what we completed for Week 8 in a simple step-by-step way.

## Week 8 Goal

The goal for Week 8 was to move from backend-only progress into the first usable frontend structure.

Before Week 8:

- the project had backend event routes and MySQL support
- the frontend was still mostly a starter structure
- there was no proper first version of the main website screens

After Week 8:

- we created the first version of the frontend
- we added a basic website structure
- we prepared the main user-facing pages such as home, events, login, and register
- we documented both the frontend screens and the current database structure

## What We Added

The main Week 8 work focused on the frontend prototype and structure.

Important folders and files:

- `navroop/Version 1/`
- `client/`
- `Docs/Week 8/`

The frontend version includes:

- a home page
- an events page
- a login page
- a register page
- a basic routing structure
- a first UI direction for the app

## Week 8 Frontend Focus

This week was mainly about getting the first visual version of the system ready.

The idea was not to finish every feature yet.

Instead, the goal was to:

- create the first frontend version
- define the overall layout and navigation direction
- show how the project will look to users
- create a base that can be connected to the backend later

## Basic Structure We Now Have

The first frontend version gives the project a visible structure.

At this stage, the system now has:

- a homepage for first impressions
- an events page for browsing campus events
- a login screen
- a registration screen
- a base design direction for later integration

This is important because it helps the team:

- visualise the final app
- review layout and user flow early
- identify what backend features still need to be connected

## Screens Completed

These are the main screens completed in the first version:

1. Home page
2. Events page

4. Register page

These screens give us the first usable frontend structure for the app.

## Database Progress Shown This Week

Week 8 also includes screenshots of the current database setup.

## What This Week Shows

By the end of Week 8, we now have:

- the first version of the frontend
- a clear basic page structure
- main user screens ready for review
- supporting screenshots of the current database structure

This means the project is now moving beyond backend-only progress and into a more complete full-stack direction.

## What Is Still Missing

Even though the first frontend version is now available, some parts are still not fully connected yet.

At this stage, the system still needs:

- real backend authentication
- real registration and bookmark logic
- full role-based access control
- stronger database support for all planned features
- full frontend-to-backend integration

That is normal for this stage of the project.

Week 8 is mainly about getting the first frontend structure ready.

## What We Learned

This week helped us understand:

- how important it is to build the UI structure early
- how frontend design can move ahead while backend logic catches up
- how page structure, user flow, and database planning all need to stay aligned
- how screenshots help document project progress clearly

## Final Result for Week 8

By the end of Week 8, the project now has a first version of the frontend with a basic structure and supporting screenshots that show both UI progress and current database progress.
